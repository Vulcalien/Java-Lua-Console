Scripting

main.lua

The console automatically calls the file 'main.lua' put in 'console-userdata/script'
This script must contain 2 functions: init() and tick()
function init()

end

function tick()

end
Both ticking and rendering operations are called in the tick() function.
init() is called before the game starts. It is strongly recommended to call initialization functions in this function to avoid possible errors.

See the Lua Documentation to learn about Console Lua Interface.


Files and folders

console-userdata

In order to use the console there must be a folder 'console-userdata' put in the same location of the .jar file.
This console will contain all the game assets and scripts.
The files that it must contain are:
  • script ~ a folder
  • sfx ~ a folder
  • config.json ~ a json file
  • atlas.png ~ a 128x128 image
  • map ~ a binary file

script

This folder contains lua files. It can also contain subfolders.
It must contain 'main.lua' (see Scripting section).

sfx

This folder contains sounds. Sounds have to be '.wav' files.

config.json

It is a json file used to store user settings. However, it has no use for now.
It should contain just two braces: {}
This will probably be removed in a future version.

atlas.png

It is a 128x128 pixels png image.
This image contains 256 sprites (8x8 pixels).

map

This is a binary file in which level data is stored.
Each byte represents a tile, and there are 8 bytes used as 'header'.
The level size is encoded in the header.
If the map is malformed, it must be replaced with a working one in order to run the game or the editor.
There is a integrated editor for creating maps.
